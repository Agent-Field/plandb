Task orchestration for AI agents.

PlanDB is a compound graph tool that helps AI agents manage complex workflows through recursive containment and cross-cutting dependencies.

Get Started

Why PlanDB?

Most task managers are either simple lists or rigid trees. PlanDB combines both into a single, powerful structure:

  • Containment (Place Graph): Tasks can contain subtasks recursively, to any depth. This provides a clean organizational structure.
  • Dependencies (Link Graph): Tasks can depend on any other task, regardless of their position in the containment tree.

This "Compound Graph" model reflects how complex work actually happens—tasks are nested for organization but linked for execution.

Built for Agents

PlanDB is designed to be the backbone of autonomous agent systems. With atomic go and done commands, it ensures coordination across multiple parallel workers without double-assignment.

$ plandb init "my-project"
$ plandb add "implement feature" --description "..."
$ plandb go
