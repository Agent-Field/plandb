CLI Reference

Complete documentation for the PlanDB command-line interface.

plandb

The main entry point for PlanDB. Use subcommands to manage projects and tasks.

Task graph primitive for AI agent orchestration.

Manages a dependency-aware task graph in SQLite. Three interfaces: CLI, MCP server, HTTP API.
Agents decompose work into tasks with dependencies, then execute via a claim-and-complete loop.

Usage: plandb [OPTIONS] [COMMAND]

init

Initialize a new project.

Usage: plandb init [OPTIONS] <NAME>

Arguments:
  <NAME>  Project name

Options:
      --description <DESCRIPTION>  Optional description of the project's goal

add

Add a new task to the current project.

Usage: plandb add [OPTIONS] <TITLE>

Arguments:
  <TITLE>  Task title

Options:
      --as <CUSTOM_ID>             Custom short ID (e.g. --as api creates t-api)
      --kind <KIND>                Task kind: generic, code, research, review, test, shell
      --description <DESCRIPTION>  Detailed description of what the task involves
      --dep <DEPS>                 Dependency: TASK_ID

go

Claim and start the next ready task.

Usage: plandb go [OPTIONS]

Options:
      --agent <AGENT>      Agent identifier (default: 'default')

done

Complete a task.

Usage: plandb done [OPTIONS] [TASK_ID]

Options:
      --result <RESULT>  Result data (JSON or plain text)
      --next             After completing, claim + start next ready task

split

Split a task into subtasks.

Usage: plandb split [OPTIONS] --into <INTO> [TASK_ID]

Options:
      --into <INTO>  Parts: comma-separated ("A, B") or chain ("A > B")

status

Show project progress.

Usage: plandb status [OPTIONS]

Options:
      --detail             Show per-task breakdown
      --full               Show all tasks and dependencies

show

Show full details of a task.

Usage: plandb show [OPTIONS] <TASK_ID>

list

List tasks with optional filters.

Usage: plandb list [OPTIONS]

Options:
      --status <STATUS>    Filter by status
      --kind <KIND>        Filter by kind

task

Advanced task management commands.

Usage: plandb task [OPTIONS] <COMMAND>

Commands:
  create        Create a new task
  list          List tasks
  get           Get task details
  go            Claim next ready task
  done          Mark task as complete
  add-dep       Add a dependency edge
  update        Update task fields
  insert        Insert a task between others

task add-dep

Add a dependency edge between two existing tasks.

Usage: plandb task add-dep [OPTIONS] --after <AFTER> <TO_TASK>

Arguments:
  <TO_TASK>  Downstream task (the one that waits)

Options:
      --after <AFTER>  Upstream task (must complete first)
