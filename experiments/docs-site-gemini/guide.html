Getting Started

This guide will walk you through the core workflow of PlanDB.

1. Initialize your project

Every PlanDB journey starts with a name.

$ plandb init "my-agent-mission"

2. Decompose the mission into tasks

A good task has a clear goal and a detailed description. Use dependencies to ensure tasks are executed in the right order.

$ plandb add "Setup project" --description "Create repo and install dependencies." --as setup
$ plandb add "Implement feature" --description "Write core logic for feature A." --dep t-setup --as feature

Note: We used --as to give tasks readable IDs (t-setup, t-feature). PlanDB generates unique IDs by default if you don't specify one.

3. Start working

When you're ready to start, use the go command. PlanDB will find the next task that is ready to run and claim it for you.

$ plandb go

4. Complete tasks and chain work

When a task is finished, mark it as done. You can use --next to automatically claim the next available task in one step.

$ plandb done --next

5. Split complex tasks on the fly

If you realize a task is too big mid-execution, split it into smaller subtasks.

$ plandb split --into "Design, Implementation, Testing"

You can also create a dependency chain when splitting:

$ plandb split --into "A > B > C"

6. Visualize progress

Check the status of your task graph at any time.

$ plandb status --detail
