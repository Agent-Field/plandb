Playground

Type PlanDB commands into a terminal-like simulator and watch the task graph evolve. Tutorial mode walks through the core workflow from project creation to task splitting and status inspection.

Tutorial mode

Follow the canonical loop: init -> add -> go -> done --next -> split -> status.

  1. 1. Initialize Create a project with plandb init "demo-flow".
  2. 2. Add work Add a first task with a full description and a custom ID.
  3. 3. Add a dependent task Create a second task that depends on the first.
  4. 4. Claim work Use plandb go to start the ready task.
  5. 5. Complete and advance Finish the current task with plandb done --next.
  6. 6. Split execution Break the running task into a dependency chain.
  7. 7. Inspect graph state Run plandb status --detail.
demo@plandb $

Supported commands

  • plandb init "name"
  • plandb add "task" --description "..." --as id --dep t-upstream --kind code
  • plandb go
  • plandb done --next
  • plandb split --into "A, B" / "A > B"
  • plandb status --detail
  • plandb show t-task
  • plandb help, clear, tutorial start

Why a simulator?

The site stays deployable as static HTML, CSS, and JavaScript while still demonstrating the mental model of the CLI. Outputs are deterministic, so examples stay coherent and tutorial mode can enforce a clean learning path.
