CLI overview

The top-level help describes PlanDB as a task graph primitive for AI agent orchestration with three interfaces: CLI, MCP server, and HTTP API. 

Usage: plandb [OPTIONS] <COMMAND>

Options:
  --db <DB>     Path to SQLite database file
  --json        Output as structured JSON
  -c, --compact Compact output optimized for LLM context windows
  -h, --help    Print help
  -V, --version Print version

Primary workflow commands

  • init creates a project.
  • add creates a task.
  • go claims the next ready task.
  • done --next completes and continues.
  • status inspects progress.
  • split decomposes a task into parts.

Additional top-level commands surfaced by help

  • use, ahead, and what-if support navigation and plan adaptation.
  • task exposes deeper task-management operations.
  • project, serve, and agent appear in the top-level command list.

Getting-started commands

These commands were individually verified with their own --help output.

plandb init

Create a project and set as default

Usage: plandb init [OPTIONS] <NAME>

Arguments: <NAME> project name.

Verified options: --description, --db, --json, -c/--compact.

plandb add

Create a new task

Usage: plandb add [OPTIONS] <TITLE>

Arguments: <TITLE> task title.

Verified task options: --project, --as, --kind, --description, --priority, --dep, --parent, --max-retries, --timeout, --requires-approval, --tag, plus the common --db, --json, and -c/--compact flags. 

$ plandb add "Implement parser" --as parser --kind code --description "Build the parser module"
$ plandb add "Write docs" --dep t-parser --description "Document parser behavior"

plandb go

Claim + start next ready task

Usage: plandb go [OPTIONS]

Verified options: --agent, --project, --db, --json, -c/--compact.

plandb done

Complete a task, optionally claim next

Usage: plandb done [OPTIONS] [TASK_ID]

Verified options: --result, --files, --next, --agent, and the common output/database flags.

plandb split

Split a task into sub-tasks

Usage: plandb split [OPTIONS] --into <INTO> [TASK_ID]

The verified --into formats are: comma-separated parts like "A, B, C", dependency chains like "A > B > C", or a JSON array.

plandb status

Show project progress: done/total, ready tasks, running agents

Usage: plandb status [OPTIONS]

Verified detail modes: status for a summary, status --detail for per-task breakdown, and status --full for all tasks plus dependency edges.

plandb show

Show full details of a task

Usage: plandb show [OPTIONS] <TASK_ID>

Use show to inspect the exact task description, status, and metadata for a specific task.

Task namespace

plandb task --help exposes the deeper task-management surface. Verified top-level usage: 

Usage: plandb task [OPTIONS] <COMMAND>

Discovered task subcommands

  • create, create-batch, list, get
  • next, go, claim, start
  • heartbeat, progress, done, fail
  • cancel, approve, add-dep, remove-dep
  • update, insert, amend, pivot
  • split, decompose, replan, pause
  • note, notes, overview

Verified dependency editing

Usage: plandb task add-dep [OPTIONS] --after <AFTER> <TO_TASK>

The verified semantics are: --after identifies the task that must finish first, and <TO_TASK> is the downstream task that receives the new dependency edge.
