PlanDB

Task graph primitive for AI agents

A compound graph model for orchestrating AI agent work — with dependency tracking, recursive decomposition, and parallel execution baked in.

Get Started CLI Reference
plandb — terminal
$ plandb init 'my-project'
Created project p-my-project
 
$ plandb add 'Design API' --kind research
$ plandb add 'Implement API' --dep t-design-api
 
$ plandb go
➔ t-design-api "Design API" running
 
$ plandb done --next

Built for composite intelligence

Everything AI agents need to coordinate work — nothing they don't.

Compound Graph

Place graph for spatial hierarchy plus link graph for typed dependencies. Two orthogonal structures that compose cleanly.

Recursive Decomposition

Split tasks mid-flight with plandb task split. Zoom into composites. Replan entire subtrees without losing state.

Multi-Agent Ready

Atomic task claiming prevents conflicts. Dependency enforcement ensures correct ordering. Multiple agents work the graph in parallel.

Zero Friction CLI

From init to done in five commands. No config files, no servers, no setup. A single SQLite file holds all state.
